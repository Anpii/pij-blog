概述#

π0.7 是 Physical Intelligence 团队最新发布的机器人基础模型，约 5B 参数量。核心亮点是：开箱即用，无需任务微调就能在大量灵巧操作任务上达到甚至超越专用 RL 微调模型的水平。

论文地址：https://pi.website/pi07

核心思路：多样化 Prompt 引导#

之前的 VLA 模型（如 π0、π0.5、π0.6）只给模型一个简短的语言指令作为上下文。π0.7 的突破性想法是：给模型提供更丰富的多模态上下文信息，包括：

  1. 子任务指令（Subtask Instructions）：不仅告诉模型"做什么"，还分步骤告诉它"接下来做什么"
  2. 子目标图像（Subgoal Images）：用生成的图像展示场景的期望状态，比文字描述更直观
  3. 片段元数据（Episode Metadata）：标注数据片段的执行速度、质量分数、是否有错误等
  4. 控制模式（Control Mode）：使用关节空间还是末端执行器控制

这个设计的巧妙之处在于——它让模型可以从质量参差不齐的数据中学习，包括失败片段和次优的自主动作数据。因为元数据告诉模型"这是个次优演示"，模型就不会简单地平均所有模式，而是学会区分好坏。

Prompt 示例#

Task: peel vegetables. Subtask: pick up the peeler. Speed: 8000. Quality: 5. Mistake: false. Control Mode: joint.

模型架构#

π0.7 构建在 π0.6 的 VLA 架构和 MEM 记忆系统之上：

  • 视觉语言主干（VLM Backbone）：基于 Gemma3 4B VLM（含 400M 视觉编码器）
  • Action Expert：860M 参数的流匹配（Flow Matching）动作生成器
  • 记忆编码器：MEM 风格的视频历史编码器，支持空间和时间压缩
  • 多模态 Prompt：语言指令 + 子目标图像 + 元数据

训练技术#

  • 知识绝缘（Knowledge Insulation）：Action Expert 的梯度不回传 VLM 主干，保持 VLM 训练的稳定性
  • Flow Matching 动作生成：预测速度场而非直接预测动作，捕捉多模态动作分布
  • 实时动作分块（RTC）：训练时模拟 0-12 步延迟，生成平滑动作轨迹
  • 随机 Dropout：训练时随机丢弃 Prompt 中的某些部分，使模型灵活适应不同测试场景

四大能力#

1️⃣ 开箱即用的灵巧操作#

π0.7 在没有任务微调的情况下，在大量灵巧任务上达到或超越了专用 RL 微调模型：

任务π0.7 表现
折叠衣物（T恤、短裤）吞吐量超越 RL 专家
制作浓缩咖啡持平 RL 专家
折叠纸箱吞吐量超越 RL 专家
切西葫芦、削果蔬接近 SFT 专家
取出垃圾袋与 SFT 专家持平

2️⃣ 指令跟随与泛化#

在 4 个未见过的厨房 和 2 个未见过的卧室 环境中测试：

  • 能执行 3-6 步的开放指令序列
  • 远超 π0.5 和 π0.6 的指令跟随成功率
  • 能理解非标准的表达方式（如"拿起我用来喝汤的东西"）
  • 能打破数据集偏见（例如在"反向清理"任务中，把本该扔垃圾的物品放进洗碗篮）

3️⃣ 跨本体迁移#

零样本跨本体迁移——这是 π0.7 最令人印象深刻的能力之一：

  • 在 UR5e 双臂机器人上折叠 T 恤，该机器人从未在训练数据中见过折叠衣物任务
  • 性能与人类遥操作专家初次尝试持平
  • 模型甚至能发现更适合目标本体的新操作策略（例如在较长手臂上使用垂直抓取而非倾斜抓取）

4️⃣ 组合任务泛化#

可以将已有技能组合成全新的任务：

  • 把红薯放进空气炸锅（训练数据中没有的操作）
  • 通过人类实时语言引导来执行新任务

消融实验#

  • 去除元数据：所有任务性能下降，特别是吞吐量
  • 去除自主动作数据：不让模型从自主评估数据中蒸馏，性能显著下降
  • 子目标图像（GC）：在复杂指令和跨本体迁移中显著提升效果

我的看法#

π0.7 代表了机器人基础模型的一个重要方向：不是通过更大的模型或更多的数据，而是通过更聪明的数据使用方式。其核心贡献在于证明了：

  1. 丰富 Prompt 设计可以让模型从次优数据中学习而不降低性能
  2. 数据多样性与详细上下文的协同效应——两者缺一不可
  3. 零样本跨本体迁移已经从实验室演示走向实用水平

与前面的 LingBot-VLA 2.0 相比，两者走了不同的路线：

  • LingBot-VLA 2.0：侧重统一动作空间表示和 MoE 架构创新
  • π0.7：侧重 Prompt 工程和数据多样化，在同样的基础架构上挖掘潜力

两者都指向同一个方向：通用机器人模型正在快速成熟。
