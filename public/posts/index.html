π0.7：可引导的通用机器人基础模型

概述 π0.7 是 Physical Intelligence 团队最新发布的机器人基础模型，约 5B 参数量。核心亮点是：开箱即用，无需任务微调就能在大量灵巧操作任务上达到甚至超越专用 RL 微调模型的水平。 论文地址：https://pi.website/pi07 核心思路：多样化 Prompt 引导 之前的 VLA 模型（如 π0、π0.5、π0.6）只给模型一个简短的语言指令作为上下文。π0.7 的突破性想法是：给模型提供更丰富的多模态上下文信息，包括： 子任务指令（Subtask Instructions）：不仅告诉模型"做什么"，还分步骤告诉它"接下来做什么" 子目标图像（Subgoal Images）：用生成的图像展示场景的期望状态，比文字描述更直观 片段元数据（Episode Metadata）：标注数据片段的执行速度、质量分数、是否有错误等 控制模式（Control Mode）：使用关节空间还是末端执行器控制 这个设计的巧妙之处在于——它让模型可以从质量参差不齐的数据中学习，包括失败片段和次优的自主动作数据。因为元数据告诉模型"这是个次优演示"，模型就不会简单地平均所有模式，而是学会区分好坏。 Prompt 示例 Task: peel vegetables. Subtask: pick up the peeler. Speed: 8000. Quality: 5. Mistake: false. Control Mode: joint. 模型架构 π0.7 构建在 π0.6 的 VLA 架构和 MEM 记忆系统之上： 视觉语言主干（VLM Backbone）：基于 Gemma3 4B VLM（含 400M 视觉编码器） Action Expert：860M 参数的流匹配（Flow Matching）动作生成器 记忆编码器：MEM 风格的视频历史编码器，支持空间和时间压缩 多模态 Prompt：语言指令 + 子目标图像 + 元数据 训练技术 知识绝缘（Knowledge Insulation）：Action Expert 的梯度不回传 VLM 主干，保持 VLM 训练的稳定性 Flow Matching 动作生成：预测速度场而非直接预测动作，捕捉多模态动作分布 实时动作分块（RTC）：训练时模拟 0-12 步延迟，生成平滑动作轨迹 随机 Dropout：训练时随机丢弃 Prompt 中的某些部分，使模型灵活适应不同测试场景 四大能力 1️⃣ 开箱即用的灵巧操作 π0.7 在没有任务微调的情况下，在大量灵巧任务上达到或超越了专用 RL 微调模型： ...
