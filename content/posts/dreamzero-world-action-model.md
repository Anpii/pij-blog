## 概述

**DreamZero** 是 NVIDIA 团队推出的 14B World Action Model（WAM），发表于 2026 年 2 月。核心洞察：**VLA（构建在 VLM 之上）只有语言先验，缺少物理动力学先验；而构建在视频扩散模型之上的 WAM 天然具备时空物理理解能力。**

论文：https://arxiv.org/abs/2602.15922 | 项目：https://dreamzero0.github.io | 代码：https://github.com/dreamzero0/dreamzero

---

## 一、为什么 VLA 不够？

当前 SOTA 机器人基础模型几乎都是 VLA（Vision-Language-Action），特点是从预训练 VLM 初始化，预测动作。VLA 的优势是继承了 VLM 的语言理解能力：

- ✅ "把可乐罐移到 Taylor Swift 那儿" —— 能理解语义
- ❌ "解开这个鞋带" —— 如果训练数据里没有就很吃力

**根本原因：VLM 预训练数据是静态图文对，只编码了"是什么"，缺乏"怎么做"的时空先验。** 运动动力学、空间精准度、几何约束——这些东西 VLM 学不到。

## 二、DreamZero 的核心思路：WAM

DreamZero 提出了一种新范式——**World Action Model (WAM)**：

```
WAM = 视频扩散模型 + 动作预测头
```

它不是从 VLM 初始化，而是从**预训练的视频扩散模型**构建：

### 架构设计

| 组件 | 说明 |
|------|------|
| **Backbone** | 14B 自回归 DiT（Diffusion Transformer） |
| **视觉编码** | VAE 编码器（视频帧 → 潜空间） |
| **语言编码** | Text Encoder |
| **状态编码** | Proprioceptive State Encoder |
| **训练方式** | 流匹配（Flow Matching） + 教师强制（Teacher Forcing） |
| **生成方式** | 逐块自回归，联合去噪视频 + 动作 |

### 联合预测：视频 + 动作

DreamZero **同时预测未来的视频帧和动作**，而不是分开处理：

```
π(video, action | observation, language, proprio)
```

这本质上将动作学习转化为**逆动力学问题**——给定预测的未来视觉状态，推断需要的动作。由于视频扩散模型已经在海量互联网视频上学到了物理常识，DreamZero 只需要额外学习：

1. 机器人视角的视频预测（微调）
2. 从视频中提取动作（逆动力学）

### 为什么选自回归而非双向？

- **KV Cache** 加速推理
- 利用历史观察引导下一步生成
- 保持原生帧率 → 视频-动作对齐更精准
- 闭环设置下，用真实观测替换生成帧 → 避免自回归误差累积

![核心公式：WAM = 视频预测 × 逆动力学]

## 三、泛化能力（实验亮点）

DreamZero 在各维度碾压 SOTA VLA：

### 1️⃣ 零样本任务泛化

在真实机器人上测试未见过的动词/动作：

| 指标 | VLA Baseline | DreamZero |
|------|:---:|:---:|
| 任务泛化 | — | **2× 提升** |
| 环境泛化 | — | **2× 提升** |
| 微调后泛化 | — | **+10%** |

可以执行的任务包括：按电梯按钮、解鞋带、用抹布擦白板、给汉堡扇风、倒酱汁到杯子里、和人类击拳……

### 2️⃣ 跨本体迁移

两种方式来迁移到新机器人：

**A. 视频数据迁移：**
- 用另一个机器人（YAM）或人类的纯视频数据
- **10-20 分钟**数据 → 未见任务性能提升 **>42%**

**B. 少样本本体适应：**
- AgiBot G1 上预训练 → YAM 上用 **30 分钟自由探索数据** 适应
- 适应后仍保持零样本泛化能力

### 3️⃣ 数据效率

打破了"需要重复演示"的传统智慧：

- 只用 **~500 小时** 真实数据
- 用**多样性**数据（不同场景、不同策略、不同质量）而非重复演示
- 甚至在没有训练过的仿真基准（Genie Sim 3.0，100 个不同任务）上也展现非平凡性能

## 四、实时推理：从 5.7 秒到 7Hz

14B 视频扩散模型做实时控制看似不可能。DreamZero 通过三级优化实现了 **38× 加速**：

### 算法级：DreamZero-Flash

解耦视频和动作的去噪步数：
- 视频：Beta(7.0, 1.0) 分布采样，偏向低噪声
- 动作：均匀分布采样
- 有效去噪步数：16 → 4

### 系统级

- **CFG 并行**：条件/无条件推理分布到 2 张 GPU → 47% 延迟降低
- **DiT 缓存**：连续速度向量余弦相似度超过阈值时复用 → 大幅减少计算

### 实现级

- **torch.compile + CUDA Graph**：消除 CPU 开销
- **后训练量化**：NVFP4（Blackwell）+ FP8 QKV/Softmax + FP16 非线性
- **cuDNN backend + GPU 调度**：消除 CPU-GPU 同步瓶颈

**结果：单张 GPU 上 5.7 秒 → 约 7Hz 实时闭环控制。**

```
等待推理 → 阻塞运动             ❌ 慢
异步执行 → 连续运动 + 并发推理    ✅ 7Hz
```

## 六、局限性

- 训练数据规模相对小（~500h），制约了更广的泛化
- 对算力要求高（14B DiT + 多 GPU）
- 在需要长期记忆的任务上未做过专项评测
- 视频质量直接决定动作质量，对世界模型要求高

## 七、我的解读

DreamZero 是机器人基础模型一条全新路径的开端，其意义不亚于 VLA 的诞生：

**1. 从"语言先验"到"物理先验"的范式转移**

VLA 证明了语言理解对机器人有用。DreamZero 证明了**物理理解（时空运动、动力学）可能更重要**——至少对新技能泛化来说是决定性的。

**2. 世界模型 = 更好的策略**

核心公式很优雅：如果你能预测世界如何变化（视频预测），那推断该做什么（动作提取）就是一个相对简单的逆动力学问题。这从根本上解释了为什么 WAM 泛化更好。

**3. 数据多样性 > 数据重复**

不需要同一个任务采集 100 遍。多样化的数据分布，配合物理先验，反而学得更好。这对数据采集策略有深远影响。

**4. 实时视频扩散**

38× 加速是从 "不能用" 到 "能用" 的关键突破。异步执行 + 多 GPU 并行 + 量化，这组合拳打得很漂亮。

**5. 开源！**

DreamZero 完全开源了模型权重、推理代码、benchmark 评测代码。这在基础模型领域是个重要的生态贡献。
